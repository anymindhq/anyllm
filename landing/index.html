The Problem

The 10-minute tax you pay every time.

You're deep in a coding session with an LLM. Then one of these happens, and the next tool needs the whole project explained again.

01

Credits Exhausted

API quota or subscription caps hit. You switch providers, but the built-up context is gone.

02

Context Window Full

The model starts forgetting earlier decisions. A fresh session does not know what happened.

03

Provider Outage

The service goes down. Re-briefing a fallback model from scratch breaks your momentum.

04

Second Opinion

You want another model to review the work, but first it needs the exact task state.

How It Works

Three commands. Thirty seconds.

Pack your current session, prime the target model, then paste a compact briefing with decisions, failed paths, and next steps.

1

Pack

anyllm pack

Reads your last LLM session and distills it into a focused snapshot with confidence ratings.

2

Prime

anyllm prime --target chatgpt

Wraps the snapshot in target-specific framing, anti-repetition guards, and verification notes.

3

Paste

--copy

The next LLM knows what is done, what failed, what matters, and what to do next.

Features

Built for mid-task handoff.

Every feature exists to make a new model resume work accurately, without pretending long-term memory is magic.

A

Cross-Provider Portability

Move context between Claude, ChatGPT, Cursor, and future LLM tools without ecosystem lock-in.

B

Confidence-Aware Briefings

Each section is rated high, medium, or low so the receiving model knows what to verify.

C

Local-First and Private

No cloud, no database, no telemetry. Transcripts and snapshots stay on disk as plain files.

D

Hand-Editable Snapshots

Snapshots are markdown. Open them, correct them, commit them, or share them like any other project file.

E

Anti-Repetition Guards

The briefing explicitly says what not to restart, re-implement, or retry.

F

Extensible Pipeline

Add new sources or targets with a small adapter while keeping the core snapshot format stable.

The .anyllm Format

A format designed to become boring infrastructure.

Versioned markdown keeps the snapshot inspectable, diff-friendly, and easy for other tools to adopt.

  • Versioned so the format can evolve without breaking old snapshots.
  • Self-contained so one file carries enough state to resume the task.
  • Confidence-aware so uncertain claims are visible at handoff time.
  • Hand-editable because plain markdown is still the right default.
.anyllm/current.md
---
anyllm_version: 0.1
project: multiagent
budget_tokens: 2000
---

# Task
Build a multi-agent orchestration layer.

# Status
Core agent loop done.
Tool router done.
State persistence in progress.

# Decisions
- Use SQLite for state. conf: high
- Event sourcing over CRUD. conf: medium

# Tried and failed
- Redis for state. Overkill. Do not redo.

# Next step
Implement persistence in store.py.
